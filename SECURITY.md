# Seguridad

No publiques documentos reales, datos personales, credenciales ni
información sensible en issues, pull requests o capturas.

## Reportar una vulnerabilidad

Para informar una vulnerabilidad, utilizá un canal privado disponible en
GitHub. Si el repositorio tiene habilitado **"Report a vulnerability"**
(pestaña *Security*), usá esa opción:

<https://github.com/CarolinaMMartin/anonimizador_uso_personal/security/policy>

No se garantiza un plazo específico de respuesta.

## Qué reportar

Interesan especialmente los reportes relacionados con:

- Lectura indebida de archivos del equipo.
- Persistencia inesperada de datos.
- Procesamiento de PDF o DOCX maliciosos.
- Llamadas de red no previstas.
- Inyección en los archivos exportados (DOCX, PDF, CSV, Markdown).

Al describir el problema, usá ejemplos sintéticos o ficticios, nunca
documentos reales ni datos personales.
